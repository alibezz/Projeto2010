Opinion Graphs for Polarity and Discourse Classification ∗

Dois nós podem ter reinforcing, non-reinforcing ou no link relations.

Discourse-level opinion graphs: os nós são opiniões, e os links são ou target links ou frame links. Cada nó tem uma polatidade (pos., neg. ou neutra) e tipo (sent. ou arg.). Os target links são rotulados como same ou alt (between targets hat are related by virtue of being opposing options in the context of the discourse). Opinion frames são de 2 tipos: reinforcing ou non-reinforcing.

3 etapas: classificar m nó, rotular o target link como same ou alt (TLC) e o frame link (FLC) como reinforcing ou non-reinforcing.

O algoritmo é assim: usa local features APENAS para predict a opinion polarity e os link types. Depois ordena randomicamente o set de todas as opiniões e links e prediz as mesmas coisas usando local feautres AND the values of the currently predicted relational features.

===> Local features: opinion lexicons, dialog information and unigram feautes. Depois, lista umas feautes que podem ser aplicadais pra TLC e/ou FLC.

=====> Links são essenciais pra ajuda a despolarizar.

Relational features: informações da vizinhança. vc minera os tipos de links e polatidades ocorrendo na vizinhança.

Experimentos: usa pedacinhos bem pequenos de discurso com a polaridade. O padrão-ouro pra target-link class, dado um DA áir, baseia-se no target link entre as targets das opiniões contidas no pair. Para o frame link, o padrão-ouro é decidido baseado no frame entre as opiniões contidas no pair. são 7 meetings; treina com 6 e testa com 1, variando isso 7 vezes.

Usa svms pra local features e outros pra relational features. ==> predizer opinion polarity using only local features o only relational features.


3 cenários: ICA-LinkNeigh, vc tem informação sobre todos os links no grafo, bem como as classes dos vizinhos (n classifica os frame links ou target links, e a polaridade n depende das predições de iterações prévias); ICA-noInfo, onde cada nodo n sabe a quais são conectados a priori e nada sabe sobre a polaridade de nenhum outro nodo (usa apenas o algoritmo iterativo com as features locais e relacionais); o ICA-LinkOnly, onde se sabe da informação sobre os links do grafo e usa-se o collective classification framework pra inferir a polaridade dos vizinhos.

Em suma, o método em si sem informação anterior sobre links ou os vizinhos funciona mto mal. Oq ele mostra eh q, colocando mais informaçao de pouco em pouco, a coisa melhora mto. Ou seja, as informaçoes da vizinhança e inks n sãoredudantes. ===> Estranho q, pra classificar os links (os FLC e os TLC separadamente), qdo era LinkOnly ia melhor às vezes que LinkNeigh. :-/
